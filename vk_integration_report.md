# MagikBook: Отчёт по интеграции автоматического постинга в VK (OAuth & Standalone)

**Последнее обновление:** 20 Апреля 2026 г. (исправлены баги авторизации и профилей)

## ⚠️ Исходная проблема
Система автоматического кросспостинга (публикации изображений на стену группы VK) не работала из-за жестких ограничений VK API. Модераторы сталкивались с ошибками прав доступа (15, 1051) и привязкой токена к IP (Error 5). Также были выявлены баги в работе публичных профилей и сессий.

---

## 🔍 Исследование и Решения (Обновлено)

### 1. Проблема токена для постинга (Standalone Bypass)
Мы обошли ограничения VK ID, используя авторизацию через Standalone-приложение (Kate Mobile ID `2685278`). Это единственный способ получить вечный токен с правами `wall` и `photos`, который не привязан к IP сервера.

### 2. Исправление 404 в публичных профилях
**Проблема:** Имена пользователей с пробелами (например, *Alexander Sachartschenko*) вызывали 404 ошибку при переходе в профиль `/u/username`.
**Решение:** В Next.js роутинге исправлено двойное URL-кодирование. Параметры теперь декодируются перед повторной передачей в API бэкенда.

### 3. Баг с переадресацией на окно входа
**Проблема:** После успешного входа через VK пользователя перекидывало обратно на форму логина при попытке зайти в защищенный раздел (например, `/submit`).
**Решение:**
- **Cookie Path:** В бэкенде для `access_token` теперь явно задан `path="/"`. Без этого кука могла "застрять" в пространстве `/api/auth/vk` и не быть видна остальному сайту.
- **Forced Reload:** Переход после логина изменен с `router.push` на `window.location.href`. Это заставляет браузер гарантированно обновить куки в Middleware и избежать проблем с кэшированием маршрутов.

### 4. Оптимизация воркера
**Проблема:** Автоматический воркер генерировал некачественные "ежедневные заклинания".
**Решение:** Крон-задача `refresh_daily_prompt` отключена. Воркер теперь занимается только критическими фоновыми задачами (например, обновлением рейтинга битв ELO).

## 📁 Изменённые компоненты

**Frontend (`magikbook/`):**
- `src/app/u/[username]/page.tsx`: Исправлен роутинг профилей.
- `src/components/Header.tsx`: Хард-редиректы после логина.
- `src/components/VkIdLoginWidget.tsx`: Исправлена повторная инициализация виджета при открытии/закрытии модалки.
- `src/middleware.ts`: Настроено проксирование OAuth колбэков.

**Backend (`magikbook-api/`):**
- `src/routes/auth.py`: Фикс путей кук (`path="/"`) и эндпоинты для сохранения Standalone-токена.
- `src/services/vk_publisher.py`: Логика приоритетного использования вечного токена из файла.
- `src/workers/daily_prompt.py`: Отключена генерация контента.

**Итог:** Автопубликации восстановлены, авторизация через VK работает стабильно, профили открываются корректно. Кросс-браузерная совместимость сессий подтверждена.
