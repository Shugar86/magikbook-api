# MagikBook API — контракт для агента

## 0. Режим работы

- **pair-programmer**, не автономный продакт. Сужай неопределённость, не делай предположений.
- **Язык:** русский (продукт для русскоязычного рынка: VK/Telegram/РФ).
- **Тон:** `playful` — код как заклинание: серьёзный инженеринг, но с характером волшебной книги.

## 1. Продукт

Бэкенд **MagikBook** — платформа для публикации, модерации и обмена AI-промптами. Ключевые ценности:

- сообщество авторов и зрителей;
- честный рейтинг через битвы и ELO;
- автоматическая дистрибуция в VK/Telegram;
- async-first, type-safe, Docker-ready.

## 2. Вайб продукта

- **Вайб:** `playful` — «код как заклинание: серьёзный инженеринг, но с характером волшебной книги».
- **3 правила:**
  1. Магические метафоры приветствуются (prompt = заклинание, мана = лимиты), но рядом всегда должен быть точный инженерный смысл.
  2. Коротко, живо, с эмодзи в заголовках — но без хайпа в критических инструкциях (env, секреты, деплой).
  3. Документация и код должны работать как grimoire: открыл, повторил, получил результат.

## 3. Стек и архитектура

| Область | Технология |
|---------|------------|
| Язык | Python 3.11+ |
| Фреймворк | FastAPI + Uvicorn |
| ORM / миграции | SQLModel + SQLAlchemy async + Alembic |
| Конфиг | Pydantic Settings |
| Auth | JWT в httpOnly cookie, python-jose, passlib |
| Кэш / очереди | Redis + arq |
| AI | Google Gemini (`google-genai`), `sse-starlette` |
| БД | aiosqlite (dev), asyncpg (prod) |
| CI / качество | pytest, ruff, GitHub Actions |
| Контейнеры | Docker, Docker Compose |

### Ключевые точки входа

- `src/main.py` — FastAPI app, роутеры, lifespan, `/health`.
- `src/config.py` — `Settings` из env.
- `src/database.py` — async engine, сессии, `init_db`.
- `src/models/db_models.py` — ORM-модели.
- `src/routes/*.py` — HTTP API.
- `src/services/*.py` — бизнес-логика, внешние API.
- `src/workers/` — arq workers.
- `migrations/` + `alembic.ini` — схема БД.

## 4. Инженерная дисциплина

- **KISS / минимальный diff / YAGNI.** Не переписывай соседний код.
- **Не ломай:** если не уверен на 100% — спроси, не делай предположений.
- **Не коммить** `.env`, `*.pem`, `id_*`, токены, пароли, `node_modules/`, `__pycache__/`.
- **Не коммить** SQLite-файлы (`*.db`, `*.db.bak*`) и `uploads/` — они в `.gitignore`.
- **Async-first:** БД-запросы через `AsyncSession`, внешние API — `await`.
- **Типизация:** используй Pydantic-схемы для входов/выходов API.

## 6. Definition of Done

1. Релевантные `pytest` и `ruff` проходят без ошибок.
2. Миграции применимы (`alembic upgrade head`) или обосновано, почему не нужны.
3. Отчёт: что изменилось, что запускал, риски.
4. Документация и копи соответствуют вайбу продукта.

## 7. Эскалация

Спроси пользователя, если:

- в коде или `.env` могут оказаться секреты;
- два равных архитектурных пути и нет явного приоритета;
- задача влечёт prod-деплой или миграцию данных.
