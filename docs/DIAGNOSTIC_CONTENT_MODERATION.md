## A. Промпты «Кинцуги» и «девушка галактика»

### Как устроена главная

Сетка «Картинки и видео» на главной — это не произвольный набор: объединяются топ по `elo_rating` с лимитом фактически **до 30** карточек (8 из `homepage` + дополнение из первой страницы `feed` с дедупом; пересечение даёт суммарно **первые ~30** позиций в сортировке по Elo).

### Найденные записи

| Что искали | Результат |
|------------|-----------|
| Кинцуги | Найден: `83bd2e8a-...`, заголовок начинается с «оживающая скульптура кинцуги…», **video**, `approved`, **elo 1668**, категория «🎨 Арты». |
| Ранг среди image+video (approved/published) | **~31-е место** по Elo. |
| Вывод | Карточка **не входит в топ-30** по Elo — с главной пропадает **из-за ранжирования**, а не из-за удаления или статуса модерации. |

| Девушка / галактика | Результат |
|---------------------|-----------|
| По «галакт» в title/prompt_text | **Записей нет.** |
| Ближайшее по «девушк» | `22eacbff-...`, «Юная девушка 16 лет…», **image**, elo **1814**, `approved` — **~21-е место**, в топ-30 **входит** (если речь о ней — на главной должна быть видна при актуальной выборке). |

Если ожидалась дословная подпись «девушка галактика», её **нет в текущей БД** — возможно другое название, удалённый промпт или другая среда.

### Битва

Пара в блоке «Битва визуализаций» подбирается **случайно**; отсутствие конкретных названий в паре — ожидаемое поведение.

### Лента `/feed`

На странице ленты порядок карточек **перемешивается на клиенте** (`Math.random()`), поэтому даже при тех же данных порядок отличается от главной.

### Кнопки «Одобрить / Отклонить»

По дизайну они есть на странице `/admin/moderation` для записей в статусе `pending`, а не на карточках общей ленты.

## B. Кнопки / ссылка «Модерация»

### Проверка таблицы `users`

Выполнено: `SELECT id, email, username, is_admin FROM users`.

**У всех пользователей `is_admin = 0` — ни одного администратора.**

В интерфейсе ссылки на `/admin/moderation` показываются только при `is_admin === true` (см. `magikbook/src/components/Header.tsx`), а `GET /api/auth/me` отдаёт флаг из БД. Поэтому **нет ссылки модерации — ожидаемо при текущих данных**.

### Назначение админа

Эндпоинт `POST /api/admin/grant/{user_id}` требует уже существующего админа (`magikbook-api/src/routes/moderation.py`) — при нуле админов первый флаг нужно выставить **напрямую в БД** (или одноразовым скриптом/миграцией), например:

```sql
UPDATE users SET is_admin = 1 WHERE id = '<user_id>';
```

После этого пользователь должен перелогиниться (или сессия подтянет актуальный `is_admin` при следующем `/api/auth/me`).

## C. Проверка в браузере (рекомендация)

- Network → `GET /api/auth/me` с того же origin: при `200` смотреть `is_admin`; при `401` — проблема сессии/cookie.
- Сравнить с SQL выше.
